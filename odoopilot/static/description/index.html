Everything runs inside Odoo -- zero external services

Unlike SaaS AI integrations, OdooPilot is a single Odoo addon. No extra server, no subscription, no data sent to a middleman.

1

Telegram -> Webhook

User sends a message. Telegram calls your Odoo's webhook endpoint (/odoopilot/webhook/telegram) with HMAC validation.

2

Identity Lookup

Odoo matches the Telegram chat ID to the linked Odoo user. The AI agent runs as that user -- inheriting their access rights automatically.

3

LLM Tool Loop

The question is sent to your chosen LLM (Claude / GPT-4 / Groq). The LLM selects the right Odoo tools and calls them via the Odoo ORM.

4

Reply + Audit

The answer is sent back to Telegram. Every action is recorded in the immutable OdooPilot audit log for compliance and review.

Security model: Each user's bot session inherits their Odoo access rights. A salesperson cannot query HR data. A regular user cannot see another user's invoices. Write actions require explicit Telegram confirmation before any Odoo record is modified.

Up and running in 5 minutes

Four steps. No server to configure. No Docker. No extra infrastructure.

1.

Install the addon

Install OdooPilot from the Odoo App Store or clone the GitHub repo into your addons path. No extra Python packages needed.

2.

Create a Telegram bot

Message @BotFather on Telegram -> /newbot -> copy the API token.

3.

Add your LLM key

Get a free or paid API key from Anthropic, OpenAI, or Groq. Groq's free tier is enough to start.

4.

Paste & activate

Go to Settings -> OdooPilot. Paste both keys, select your LLM, click "Register Webhook". Done -- chat immediately.

Team onboarding: Each team member messages the bot -> taps a one-time magic link -> their Odoo account is linked. The whole team is live in under 10 minutes. No IT intervention required.

Frequently asked questions

  • Does this work with Odoo 17 Community Edition?
    Yes. OdooPilot was built specifically for Odoo 17 Community. It does not require Odoo Enterprise. It depends only on the mail, base_setup, and web modules that ship with every Odoo install.
  • Do I need all Odoo apps installed?
    No. OdooPilot gracefully detects which modules are installed. If you don't have CRM, the CRM tools are silently skipped. If you don't have HR, leave tools won't appear. Only the tools relevant to your installed apps are active.
  • Is my data sent to a third party?
    Only the user's question and the Odoo data retrieved by the tools are sent to your chosen LLM provider (Anthropic, OpenAI, or Groq). Nothing is sent to OdooPilot's servers -- because there aren't any. OdooPilot is a standalone Odoo addon with no cloud backend.
  • Can one Telegram user access another user's data?
    No. The AI agent runs as the linked Odoo user and inherits their access rights from Odoo's standard security model. A salesperson cannot see HR records they wouldn't see in the Odoo UI.
  • Which LLM is recommended?
    For production use, Anthropic Claude 3.5 Haiku offers the best balance of speed, accuracy, and cost (~$0.001 per conversation). For testing or low-budget deployments, Groq's free tier (Llama 3.3 70B) works well with no API cost.
  • Is there a hosted / SaaS version?
    No. OdooPilot is self-hosted only -- it runs entirely inside your Odoo instance. This keeps your data private and your costs minimal. There are no subscription fees, seat fees, or usage charges from OdooPilot itself.
  • What happens when the Odoo server restarts?
    Conversation history is stored in the Odoo database (not memory), so it persists across restarts. Active AI requests in-flight during a restart may time out and show a retry message to the user.
  • Can I add custom tools for my own Odoo models?
    Yes. The tool registry in services/tools.py is designed to be extended. Add a function, register it in TOOL_DEFINITIONS, and it's immediately available. The module is fully open-source (LGPL-3) -- fork and customise freely.
