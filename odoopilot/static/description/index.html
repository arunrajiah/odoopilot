Everything runs inside Odoo — zero external services

Unlike SaaS AI integrations or modules that require a separate Python service, OdooPilot is a single self-contained Odoo addon. No extra server, no Docker, no subscription, no data sent to a middleman cloud.

1

Webhook delivery

User sends a Telegram or WhatsApp message. The platform calls your Odoo webhook endpoint with HMAC or token validation. Processed asynchronously in a background thread.

2

Identity & session lookup

Odoo matches the chat ID to the linked Odoo user. The AI agent runs as that user — inheriting their Odoo access rights automatically. No privilege escalation possible.

3

Agentic LLM tool loop

The question is sent to your chosen LLM (Claude / GPT-4 / Groq). The LLM selects the right Odoo ORM tools in a multi-turn agent loop. Read tools execute immediately; write tools pause for inline confirmation.

4

Reply + audit log

The answer is sent back to Telegram or WhatsApp. Every tool call is recorded in the immutable OdooPilot audit log — user, tool, args, result, timestamp.

Security model: Each user's bot session inherits their Odoo access rights. A salesperson cannot query HR data they wouldn't see in the Odoo UI. A regular user cannot see another user's invoices. Write actions require explicit inline confirmation before any Odoo record is modified. API keys are stored in Odoo's system parameters, never exposed.

Up and running in 5 minutes

No server to configure. No Docker. No extra Python packages. No infrastructure.

1

Install the addon

Copy odoopilot/ into your Odoo addons path and run -u odoopilot, or install directly from the Odoo App Store. No extra pip packages required.

2

Create your bot

For Telegram: message @BotFather → /newbot → copy token. For WhatsApp: create a Meta app and note your Phone Number ID and Access Token.

3

Add your LLM key

Get an API key from Anthropic, OpenAI, or Groq. Groq's free tier is enough to start — no credit card needed.

4

Paste & activate

Go to Settings → OdooPilot. Paste bot credentials and LLM key, click "Register Webhook". Done — start chatting immediately.

Team onboarding in under 10 minutes: Each employee sends /link to the bot, taps the one-time magic link while logged into Odoo, and their account is linked. The whole team is live without any IT intervention.

Frequently asked questions

  • Does this work with Odoo 17 Community Edition?
    Yes. OdooPilot was built specifically for Odoo 17 Community. It does not require Odoo Enterprise. It depends only on mail, base_setup, and web — modules that ship with every Odoo install.
  • Does it work on Odoo Online / Odoo.sh?
    Odoo.sh: yes — it is a standard addon, installs like any other. Odoo Online (odoo.com SaaS): unfortunately not, because Odoo Online does not allow custom addon installation. OdooPilot requires a self-hosted or Odoo.sh instance that is reachable from the internet for webhook delivery.
  • Which messaging channels are supported?
    Both Telegram (via Bot API webhook) and WhatsApp (via Meta's WhatsApp Cloud API) are supported with full feature parity — including the /link identity flow, write-action confirmations, and proactive push notifications. Both channels can be active simultaneously.
  • Which AI / LLM providers are supported?
    Anthropic Claude (claude-3-5-haiku, claude-opus-4, claude-sonnet-4), OpenAI ChatGPT / GPT-4 (gpt-4o, gpt-4o-mini, gpt-4-turbo), Groq (llama-3.3-70b — free tier), and any OpenAI-compatible endpoint including Ollama for fully local/private AI. No SDK dependencies — provider APIs are called via raw requests.
  • How does multi-language support work?
    Each user can send /language fr (or any supported ISO 639-1 code) in Telegram or WhatsApp to set their preferred reply language. OdooPilot stores the preference in the Odoo identity record and instructs the LLM to always reply in that language. Send /language auto to revert to auto-detect. Admins can set language per user in the Odoo backend (Settings → OdooPilot → User Identities).
  • Do I need all Odoo apps installed?
    No. OdooPilot gracefully detects which modules are installed. If you don't have CRM, the CRM tools are silently skipped. If you don't have HR, leave tools won't appear. Only tools relevant to your installed apps are active — no errors, no configuration required.
  • Is my Odoo data sent to a third party?
    Only the user's question and the Odoo data retrieved by the AI tools are sent to your chosen LLM provider (Anthropic, OpenAI, or Groq). Nothing is sent to OdooPilot's servers — because there aren't any. OdooPilot is a standalone Odoo addon with zero cloud backend. For full privacy, use Ollama (local LLM) — then no data leaves your server at all.
  • Can users access data they shouldn't see?
    No. The AI agent runs as the linked Odoo user and inherits their Odoo access rights from Odoo's standard security model. A salesperson cannot see HR records they wouldn't see in the Odoo UI. A linked employee cannot see another employee's private data.
  • Which LLM is recommended for production?
    For the best accuracy and speed, Anthropic Claude 3.5 Haiku (~$0.001 per conversation). For zero API cost, Groq free tier (Llama 3.3 70B) works well. For complete data privacy, Ollama runs a local model — no data leaves your server.
  • Can I add custom tools for my own Odoo models?
    Yes. The tool registry in services/tools.py is designed to be extended. Add a function, register it in TOOL_DEFINITIONS, and it's immediately available in the AI agent. The module is fully open-source (LGPL-3) — fork and customise freely.
